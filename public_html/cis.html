0.3 Big-O Notation

We want to characterize the efficiency of an algorithm so that it is machine independent We will display the runtime as a function of the size of the input Let f(n) and g(n) be functions from positive integers to positive reals. We say that f = O(g) (which means f grows no faster than g) if there is a constant c > 0 such that f(n) <= c g(n) The function that scales better as input grows is the superior function Can show superiority of function by dividing it by another function, if it reduces to a real number the top function is superior Simplifying Rules
  1. Multiplicative constants can be omitted
  2. n^a dominates n^b if a > b
  3. Any exponential dominates any polynomial, 3^n dominates n^5, and also dominates 2^n
  4. Any polynomial dominates any logarithm, n dominates (log n)^3, and n^2 dominates n log n
